# 启动脚本自动更新功能说明

## 📋 功能概述

启动脚本现在支持**自动从 GitHub 拉取最新代码**，确保每次运行时使用最新版本。

---

## 🎛️ 开关控制

### 启用自动更新（默认）

编辑启动脚本，设置：
```bash
AUTO_UPDATE=true
```

### 禁用自动更新

编辑启动脚本，设置：
```bash
AUTO_UPDATE=false
```

---

## 📍 配置位置

### Mac 端
编辑 `start_mac.sh` 文件的第 9 行：
```bash
# ============ 配置选项 ============ #
# 自动更新开关：true=启用自动更新，false=禁用自动更新
AUTO_UPDATE=true
# ================================= #
```

### 树莓派端
编辑 `start_pi.sh` 文件的第 9 行：
```bash
# ============ 配置选项 ============ #
# 自动更新开关：true=启用自动更新，false=禁用自动更新
AUTO_UPDATE=true
# ================================= #
```

---

## 🔄 工作流程

### 1. 启用自动更新时

```
启动脚本
    ↓
检查是否是 Git 仓库
    ↓
检查是否有未提交的本地修改
    ↓
├─ 有未提交修改 → 跳过更新，显示警告
└─ 无未提交修改 → 执行 git pull
    ↓
检查依赖文件是否变化
    ↓
├─ 有变化 → 提示重新安装依赖
└─ 无变化 → 继续启动程序
```

### 2. 禁用自动更新时

```
启动脚本
    ↓
显示"自动更新已禁用"提示
    ↓
直接启动程序
```

---

## ✅ 智能检测功能

### 1. 本地修改检测
- 自动检测是否有未提交的本地修改
- 如有修改，跳过更新以避免冲突
- 提示用户先提交或暂存修改

### 2. 依赖变化检测
- 检测 `pyproject.toml`、`setup.py`、`requirements.txt` 是否变化
- 如有变化，提示用户重新安装依赖
- 避免因依赖不匹配导致的运行错误

### 3. Git 仓库检测
- 检测当前目录是否是 Git 仓库
- 非 Git 仓库时跳过更新

---

## 📊 使用场景

### 推荐启用自动更新的场景
- ✅ 多人协作开发
- ✅ 频繁更新代码
- ✅ 需要保持最新功能
- ✅ 生产环境部署

### 推荐禁用自动更新的场景
- ⚠️ 正在开发调试
- ⚠️ 网络不稳定
- ⚠️ 需要使用特定版本
- ⚠️ 有未提交的本地修改

## 🔧 手动更新方法

如果禁用了自动更新，可以手动更新代码：

### Mac 端
```bash
cd ~/Repository/lerobot_alohamini
git pull origin main
pip install -e ".[feetech]"  # 如有依赖变化
```

### 树莓派端
```bash
cd ~/lerobot_alohamini
git pull origin main
pip install -e ".[feetech]"  # 如有依赖变化
```

## ⚠️ 注意事项

### 1. 网络要求
- 需要能够访问 GitHub
- 建议在网络稳定时启用自动更新

### 2. 权限要求
- 需要有 Git 仓库的读取权限
- SSH 密钥需要正确配置

### 3. 本地修改
- 有未提交修改时会跳过更新
- 建议定期提交或暂存本地修改

### 4. 依赖安装
- 依赖文件变化时需要手动重新安装
- 使用 `pip install -e ".[feetech]"`

## 🐛 故障排查

### 问题 1：更新失败
**可能原因**：
- 网络连接问题
- Git 权限问题
- 远程仓库不可访问

**解决方法**：
```bash
# 检查网络连接
ping github.com

# 检查 Git 配置
git remote -v

# 手动拉取测试
git pull origin main
```

### 问题 2：提示有本地修改
**可能原因**：
- 有未提交的文件修改
- 有未追踪的新文件

**解决方法**：
```bash
# 查看修改状态
git status

# 提交修改
git add .
git commit -m "描述修改内容"

# 或暂存修改
git stash
```

### 问题 3：依赖安装失败
**可能原因**：
- pip 源问题
- 依赖冲突

**解决方法**：
```bash
# 使用默认源
pip install -e ".[feetech]" --index-url https://pypi.org/simple

# 或清理缓存重装
pip cache purge
pip install -e ".[feetech]"
```
